# VeritasCarbon-ESG-35K

**A large-scale, traceable ESG instruction dataset generated by a multi-agent Council of Domain Experts (CoDE).**

- **Paper**: VeritasCarbon: A Scalable Multi-Agent Framework for Generating Traceable ESG Instruction Data (VLDB 2027)
- **GitHub**: https://github.com/YihanJIANG-lab/VeritasCarbon-VLDB2027
- **License**: CC BY-SA 4.0
- **Language**: Chinese (instructions), Chinese/English (responses depending on task type)

## Dataset Summary

VeritasCarbon-ESG-35K contains **35,009 high-quality instruction-response pairs** derived from **17,721 ESG disclosure documents** spanning four corpus layers:

| Layer | Description | Documents | Chunks |
|-------|-------------|-----------|--------|
| Layer 1 | Domain textbooks & standards | 97 | 8,877 |
| Layer 2 | Corporate CSR/Sustainability reports (2006–2024) | 17,425 | 5,395 |
| Layer 3 | Regulatory guidelines & policies | 92 | 3,237 |
| Layer 4 | Industry analyses & research papers | 107 | 2,491 |

Every pair retains **full provenance** (source chunk ID, document ID, generation metadata) and a **quality score** (0–1) assigned by the CoDE MetaExpert.

## Data Fields

Each record contains:

- `instruction`: The generated task instruction (Chinese)
- `input`: The source text chunk from the ESG document (context)
- `output`: The generated response (Chinese or English)
- `source_chunk_id`: Unique identifier linking back to the source corpus chunk
- `metadata`: Rich provenance and generation metadata:
  - `quality_score`: Float in [0, 1] (MetaExpert assessment)
  - `selected_experts`: List of expert agents used (e.g., `["analysis_expert", "evaluation_expert"]`)
  - `collaboration_mode`: `"parallel"` or `"sequential"`
  - `max_iterations`: Number of feedback rounds (0–2)
  - `knowledge_items_count`: Number of injected domain knowledge items
  - `core_topics`: Identified ESG sub-topics
  - `model`: Generator model (`"Qwen2-72B-Instruct"`)
  - `framework`: `"CoDE"`

## Generation Pipeline

The dataset was generated by **VeritasCarbon**, a multi-agent framework with the following pipeline:

1. **Document Ingestion**: Parse 17,721 ESG documents (PDF/TXT) across 4 layers
2. **Semantic Chunking**: Split into 20,000 semantically coherent chunks
3. **Expert Selection**: Rule-based selector assigns 1+ of 11 expert types per chunk
4. **Knowledge Injection**: Retrieve domain-specific ESG knowledge items
5. **Multi-Expert Collaboration**: Parallel or sequential generation by assigned experts
6. **MetaExpert Feedback**: Iterative quality refinement (up to 2 rounds)
7. **Quality Scoring**: Automatic assessment on correctness, completeness, relevance, structure

All hyperparameters and random seeds are fixed (`seed=42`). See the [GitHub repository](https://github.com/YihanJIANG-lab/VeritasCarbon-VLDB2027) for the full generation code.

## Evaluation

The dataset was evaluated against three baselines (Direct Prompting, Self-Instruct, WizardLM-Evol) on 7 intrinsic metrics:

| Metric | CoDE (ours) | Best Baseline | Improvement |
|--------|-------------|---------------|-------------|
| ROUGE-L | 0.3380 | 0.0838 | **4.0×** |
| BLEU-4 | 0.1932 | 0.0424 | **4.6×** |
| Domain Relevance | 0.3637 | 0.0249 | **14.6×** |
| FactCheck Score | 0.9478 | 0.9312 | — |

Evaluation used a fixed 2,000-pair subset (`random.seed(42)`) for fair comparison.

## Limitations

- **Language**: Primarily Chinese; some responses may contain English terms common in ESG reporting
- **Domain specificity**: Optimized for ESG/finance; transfer to other domains requires adaptation
- **Temporal coverage**: Source documents span 2006–2024; regulations and standards evolve
- **Copyright**: Source documents are copyrighted by their respective issuers; only generated instruction-response pairs and metadata are released
